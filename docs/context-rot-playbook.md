# Context Rot 実践プレイブック

長セッションで序盤の指示が attention 薄れにより無視される現象（Context Rot）への対処ガイド。token-analyzer の数値を見たら、このページのアクションに従う。

## 鉄則

1. **context は使い捨て** — 記憶は外部化（MEMORY.md / Issue / PR / ファイル）
2. **読ませず参照させる** — 全文を Read させず「ファイル名・行番号だけ伝える」
3. **サブエージェントは context の防波堤** — 親 context を汚さない

## シーン別対処

| シーン | やるべきこと | やってはいけない |
|---|---|---|
| セッション開始時 | 1タスク=1セッション宣言。ゴールを最初に明示 | 「ついでにあれも」で複数タスクを混ぜる |
| 30分 or 50k token 経過 | `python3 -m src.cli` で使用率確認 | 体感で続行 |
| 使用率 70% 超え | `/checkpoint` → `/clear` → 新セッションで再開 | そのまま続行して精度劣化を放置 |
| 大きなファイル探索 | Agent tool (Explore subagent) に逃がす | 親 context で grep 連打 |
| 長い調査結果 | サブエージェントに要約だけ返させる | 全文を親に取り込む |
| 長いログ・エラー | ファイルに書いて「N行目を見て」と指示 | 全文をプロンプトにペースト |
| 完了タスクの会話 | MEMORY.md に成果だけ書いて忘れる | スクロールして過去発言を参照 |
| モデル切り替え | 重い思考だけ Opus、流れ作業は Sonnet/Haiku | 全部 Opus |

## 計測 → アクション早見表

| token-analyzer 表示 | アクション |
|---|---|
| < 50% | そのまま継続 |
| 50–70% | 不要な読み込みを止める。残タスクを絞る |
| 70–85% | checkpoint → 新セッション準備 |
| > 85% | 即 `/clear`。継続は精度劣化を招く |

## サブエージェント活用パターン

- **探索**: 「`src/` で X を実装している箇所を探して、ファイル:行 のリストだけ返して」
- **要約**: 「この PR の差分を読んで、変更点を箇条書き5行で返して」
- **検証**: 「テストを実行して、失敗したテスト名と1行サマリだけ返して」

いずれも **戻り値を最小化** することで親 context の消費を抑える。

- MEMORY.md: 長期記憶の外部化先
- `/checkpoint` `/clear`: セッション分割の標準操作
